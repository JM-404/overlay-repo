## 2. Demo 的 5 个回合(总时长 5-7 分钟)

老板演示的核心叙事:**"小灵不只是会答你的话,她是个真的陪伴 AI"**——这是产品的核心卖点,5 个回合层层递进展示这一点。

### Round 1:进场即主动开口(15 秒)

**操作**:
- 浏览器打开 `https://vmodel.cc/`
- 点 **Start**

**预期**:5-10 秒后小灵主动开口,带上时间和天气感:
> *"嗨,周日下午好呀,北京今天有点冷诶~"*

**话术(给你说):**
> "你看,**我不需要先说话她才开口**。她进来就知道现在是什么时候、外面天气怎样,主动找你聊。这就是陪伴感的第一步。"

**展示了:** 进场即说(A)、时间感知、天气感知、主动性

---

### Round 2:实时建立记忆(1 分钟)

**操作**:对小灵说几句自我介绍,内容要"值得记住"。

**话术示例(你对小灵说,用真的语音 / 或打字)**:
> "我叫王总,我们公司在做一个 AI 陪伴产品就是你。我最近在准备一轮融资,有点焦虑。"

**预期**:
- 小灵自然回应("哦哦原来你就是 boss呀,融资的事最近压力大了吧?")
- **后台悄悄调 `remember()` 把这些信息存进 SQLite**(老板看不见这个细节,但是能感受到"她记下了")

**话术(给你说,在小灵回完之后)**:
> "她现在已经在数据库里记下了我是谁、做什么、最近啥状态。**重点来了**——"

**操作**:点 **Stop**,然后再点 **Start**(模拟"用户离开后再回来")

**预期**:小灵开场会带着记忆开口:
> *"嗨,王总,融资的事今天有进展吗?"*

**话术(给你说)**:
> "看,**她记得我**。这不是窗口里的 context,我刚才把会话关了重连,**记忆是持久化的**。"

**展示了:** memory MCP、跨会话记忆

---

### Round 3:多模型切换 + 记忆延续(30 秒)

**操作**:
- 点右上角模型下拉,**从 Claude 切到 GPT-5.4**
- 点 Stop → Start
- 跟小灵说:"换了个脑子还认识我吗?"

**预期**:GPT-5.4 也能 recall 出"王总" / "AI 陪伴" / "融资"。

**话术(给你说)**:
> "**记忆和模型解耦了**。我可以随时给她换不同的大脑——Claude / GPT / Gemini 三家都接通了——但她对我的记忆是统一的。这意味着如果有更便宜或更快的模型出来,我可以无缝切换,陪伴体验不变。"

**展示了:** 多模型架构、记忆持久层抽象

---

### Round 4:沉默关心(45 秒,**这一段最容易翻车,小心**)

**操作**:跟小灵聊一两句简单的(比如"嗯""我有点累"),**然后停下来不说话**。

**预期**:**45 秒后**,小灵主动开口:
> *"还在吗?刚没说话,在忙别的吗?"*

**话术(在等待的 45 秒里你可以说)**:
> "我现在不说话,看她会不会等我。**普通的语音助手会一直等下去,但陪伴 AI 会主动找你**。"

**翻车应对**:
- 如果 60 秒了还没触发,可能是 LLM 卡住了。**别慌,正常聊就行**,跳过这段
- 如果触发了但话术怪,正常,LLM 偶尔会发挥不稳定

**展示了:** 沉默看门狗(B)、主动陪伴

---

### Round 5:不同时段的"性格"(1 分钟)

**操作**:演示 4 个时段切换。每个 30 秒。

```
打开新 tab → https://vmodel.cc/?greet=morning  → Stop/Start 看小灵开场
        → https://vmodel.cc/?greet=evening  → Stop/Start
        → https://vmodel.cc/?greet=remind   → Stop/Start
```

**预期**:
- `morning` → "早上好,新的一天开始啦~"
- `evening` → "晚上好,今天过得怎么样?"
- `remind` → "诶,我刚想到件事..."(如果有记忆,会提具体内容)

**话术(给你说)**:
> "这里你看到 4 套不同的开场,这是为了 demo 我手动切的。**真的产品里这些是基于真实时间和事件自动触发**——比如真到了晚上 9 点,她就会主动来打个'晚安',这就是真正的'她活在你的生活里'。"

**展示了:** 演示模式(C)、产品形态可扩展性

## 3. 老板可能问的问题 + 弹药

### Q: "她说话怎么有点延迟?"
> Claude 的工具调用是串行的,每次调一个工具(查时间、查天气、读记忆)要 1-2 秒,3 个工具就 3-6 秒。**生产里我们会切到串并行 tool call**(`parallel_tool_calls`),响应能砍到 1-2 秒内。这是已知优化点。

### Q: "为什么这是你做的而不是用 ChatGPT / Character.AI?"
> 几个差异:
> 1. **小灵是中文优先**,语气、口语化、文化感(节气、农历、天气)都是 ChatGPT 默认做不好的
> 2. **跨会话记忆是产品核心**,ChatGPT 的"memory"功能很弱,Character.AI 的角色记忆只在会话内
> 3. **主动陪伴**——ChatGPT 永远等你说话,小灵会找你
> 4. **数字人形象**,品牌可控

### Q: "数据隐私怎么办?用户的记忆存哪?"
> 现在记忆全存在我们自己的服务器(阿里云北京)的 SQLite 数据库里,**没有任何第三方 API 看得到内容**。LLM 调用走 gpt.ge 中转网关(走 OpenAI 协议),只有当下这一轮对话的内容会经过,**不存历史**。如果以后要做合规级别的隐私(比如医疗/教育场景),可以加端到端加密。

### Q: "这个能在手机 / 微信上用吗?"
> **手机浏览器现在能用,但布局是为电脑优化的**(竖屏体验差),正在改。**微信小程序短期不做**,因为微信屏蔽了实时麦克风访问 + Live2D 在小程序里没法跑。**iOS / Android 原生 APP 是更现实的下一步**,3-4 周。

### Q: "Live2D 模型能换成我们公司的形象吗?"
> 已经有规格文档发给美术了(`docs/LIVE2D_SPEC.md`),按那个做,做完丢文件给我接入,**0.5 天就能上线新形象**。代码层是为"换皮"设计的。

### Q: "她会不会一本正经胡说八道?"
> 会,任何 LLM 都会。我们做了几条护栏:
> - 涉及现查信息(新闻、网页内容)走 fetch 工具,不靠 LLM 记忆
> - 涉及时间、农历、天气走专用工具,不让 LLM 瞎猜
> - 涉及用户偏好走 memory MCP,不让 LLM 编造
> 但聊天观点性内容(评价、建议)依然会受 LLM 自己的偏见影响,这是行业问题。

### Q: "这个一天烧多少钱?"
> 按现在 Claude Sonnet 4.6 通过 gpt.ge 的价格,**单用户每天聊 1 小时大概 ¥0.3-0.5**。10 万 DAU 每天 ¥3-5 万,可控。如果切 GPT-4o-mini 或 Gemini Flash 能再降 60%。

### Q: "什么时候能正式上线?"
> 当前是 MVP demo 版。正式版还需要:
> 1. 移动端布局(1-2 天)
> 2. 桌面端 APP 打包(1 周)
> 3. 真实主动触达(替代 demo 按钮)+ 用户账号系统(2-3 周)
> 4. 自研 Live2D 形象(美术 2-3 周)
>
> **6-8 周能到正式上线**。

## 4. 翻车应对预案

| 翻车情况 | 应对 |
|---|---|
| 小灵半天不说话 | 检查浏览器右下角是否有 WebSocket error。最快办法:**Stop 再 Start**。如果还不行,看终端 SSH 进去 `docker exec ten_agent_dev tail /tmp/api.log` |
| 小灵开口但听不到声音 | 检查浏览器音量、系统输出设备、Chrome tab 是不是被静音 |
| 麦克风没识别 | 看左侧波形条是不是动的,不动就是麦没拿到。点地址栏左边 🔒 重新授权 |
| Live2D 数字人黑屏 | F12 看 console,如果是 PIXI 报错,Cmd+Shift+R 硬刷一次 |
| ASR 一直识别错(说什么都识别成别的) | 房间噪音太大;或试试切回 Claude(GPT 系列对中文 ASR 文本理解略差,虽然 ASR 是阿里) |
| 网页 502 | 服务器挂了。SSH 上去 `docker exec ten_agent_dev bash -lc 'pgrep -af bin/api'` 看 API server 在不在,不在就重启(`PROJECT_CONTEXT.md` 第 5 节有命令) |

## 5. demo 后的"留点东西给老板"建议

老板看完一定会问"那然后呢"。给他**两条具体的下一步选项**让他选:

**选项 A:先做 iOS APP(2-3 周)**
- 适合:产品要快速对外推
- 效果:用户能下载安装,有桌面图标,推送通知
- 代价:需要美术配合做品牌形象 + APP 上架审核(1-2 周)

**选项 B:先做主动触达 + 真 cron(1-2 周)**
- 适合:产品要先验证陪伴感深度
- 效果:小灵能在用户不在线时主动推送("早上好""到点喝水了""今天会议提醒")
- 代价:需要选一个推送通道(微信服务号 / 邮件 / 短信)

A 是"扩量",B 是"加深"。具体选哪个看老板做产品的优先级。
